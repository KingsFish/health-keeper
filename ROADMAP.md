## 优化点清单

### AI 功能优化

| 优先级 | 问题 | 建议方案 | 状态 |
|--------|------|----------|------|
| 高 | OCR 识别不完整 | 优化 prompt，针对医疗文档格式优化 | 待做 |
| 高 | 提取字段不完整 | 改进 JSON Schema，添加更多医疗字段 | 待做 |
| 中 | 无批量处理 | 添加批量 OCR/提取功能 | 待做 |
| 中 | 无重试机制 | 添加失败自动重试 (3次) | 待做 |
| 低 | 无进度反馈 | 添加流式输出支持 | 待做 |

**待添加的医疗字段：**
- 检查项目列表
- 生命体征 (血压、体温、心率等)
- 过敏信息
- 既往病史
- 家族病史

### Web UI 优化

| 优先级 | 问题 | 建议方案 | 状态 |
|--------|------|----------|------|
| 高 | 无图片预览 | 添加图片预览组件 (点击放大) | 待做 |
| 高 | 提取结果展示粗糙 | 独立卡片展示各字段，支持编辑保存 | 待做 |
| 高 | 无状态反馈 | 添加 loading 动画和进度条 | 待做 |
| 中 | PDF 预览 | 添加 PDF 内嵌预览 | 待做 |
| 中 | 无历史记录 | OCR/提取历史版本管理 | 待做 |
| 低 | 无暗色模式 | 添加深色主题支持 | 待做 |

### 性能优化

| 优先级 | 问题 | 建议方案 | 状态 |
|--------|------|----------|------|
| 中 | 图片未压缩 | 上传前自动压缩 (>2MB 的图片) | 待做 |
| 中 | 无缩略图 | 生成缩略图加速列表加载 | 待做 |
| 低 | 同步请求阻塞 | 改用异步任务队列 | 待做 |
| 低 | 无缓存 | 添加 API 响应缓存 | 待做 |

### 数据安全

| 优先级 | 问题 | 建议方案 | 状态 |
|--------|------|----------|------|
| 中 | 无加密 | 敏感字段加密存储 (AES-256) | 待做 |
| 中 | 无备份 | 自动备份机制 (每日增量备份) | 待做 |
| 低 | 无权限控制 | 多用户权限管理 | Phase 4 |

## 开发计划

### Phase 2: 功能完善 (预计 1-2 周)

**目标：优化用户体验，完善核心功能**

```
Week 1:
├── [高] 提取结果页面优化
│   ├── 结构化展示各字段
│   ├── 支持编辑和保存
│   └── 添加历史版本
│
├── [高] 图片预览功能
│   ├── 点击放大
│   ├── 旋转/缩放
│   └── PDF 预览
│
└── [高] 状态反馈
    ├── Loading 动画
    ├── 进度条
    └── 错误提示

Week 2:
├── [中] 批量处理功能
│   ├── 批量 OCR
│   ├── 批量提取
│   └── 进度显示
│
├── [中] 数据导出
│   ├── 导出 PDF 报告
│   ├── 导出 Excel
│   └── 导出 JSON
│
└── [中] AI 优化
    ├── 改进 OCR prompt
    ├── 添加更多提取字段
    └── 失败重试机制
```

### Phase 3: 多端支持 (预计 2-4 周)

**目标：开发移动端和同步服务**

```
Week 3-4: Android App
├── 项目搭建 (React Native / Flutter)
├── 拍照上传功能
├── 相册导入
├── 离线缓存
└── 与服务器同步

Week 5-6: macOS App
├── Tauri 桌面应用
├── 扫描仪集成
├── 本地存储优化
└── 菜单栏快捷操作

Week 7-8: 同步服务
├── 同步服务开发
├── 变更检测
├── 冲突合并
├── 离线队列
└── Docker 部署方案
```

### Phase 4: 商业化 (按需)

**目标：产品化，支持商业化运营**

```
├── 用户系统
│   ├── 注册/登录
│   ├── 多租户支持
│   └── 权限管理
│
├── 数据安全
│   ├── 端到端加密
│   ├── 数据审计
│   └── 合规认证
│
├── 云端部署
│   ├── 公有云部署
│   ├── 自动扩缩容
│   └── 监控告警
│
└── 商业模式
    ├── 免费版 (本地)
    ├── Pro 版 (云同步)
    └── 企业版 (私有部署)
```
