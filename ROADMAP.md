## 优化点清单

### 用户需求（头脑风暴 2026-03-25）

| 优先级 | 功能 | 说明 | 状态 |
|--------|------|------|------|
| 高 | 发票/影像"仅归档"模式 | 上传时可选跳过 OCR，仅保留原件。适用：发票、CT/MRI 影像 | 待做 |
| 高 | 检验结果展示 | 前端展示 AI 提取的检验数值（数据模型已有 LabResult），显示名称、数值、单位、参考范围、异常标记 | 待做 |
| 中 | 批量录入优化 | 上传多张图片时智能识别分组，自动创建多条就诊记录 | 待做 |
| 中 | 数值趋势图 | 选择某个指标（血糖、血压等）查看历史趋势，可视化图表展示 | 待做 |
| 低 | 时间范围筛选 | 搜索增强：按"最近3个月"、"去年"等时间范围筛选 | 待做 |
| 后续 | 数据备份 | 自动备份机制 | 待做 |
| 后续 | 移动端 | 手机拍照直接上传 | 待做 |
| 后续 | 分享链接 | 生成临时访问链接（需云服务支持） | 待做 |

### AI 功能优化

| 优先级 | 问题 | 建议方案 | 状态 |
|--------|------|----------|------|
| 高 | OCR 识别不完整 | 优化 prompt，针对医疗文档格式优化 | 待做 |
| 高 | 提取字段不完整 | 改进 JSON Schema，添加更多医疗字段 | 待做 |
| 中 | 无批量处理 | 添加批量 OCR/提取功能 | 待做 |
| 中 | 无重试机制 | 添加失败自动重试 (3次) | 待做 |
| 低 | 无进度反馈 | 添加流式输出支持 | 待做 |

**待添加的医疗字段：**
- 检查项目列表
- 生命体征 (血压、体温、心率等)
- 过敏信息
- 既往病史
- 家族病史

### Web UI 优化

| 优先级 | 问题 | 建议方案 | 状态 |
|--------|------|----------|------|
| 高 | 无图片预览 | 添加图片预览组件 (点击放大) | ✅ |
| 高 | 提取结果展示粗糙 | 独立卡片展示各字段，支持编辑保存 | ✅ |

| 低 | 无暗色模式 | 添加深色主题支持 | 待做 |

## 开发计划

### Phase 2: 功能完善 (预计 1-2 周)

**目标：优化用户体验，完善核心功能**

```
Week 1:
├── [高] 提取结果页面优化
│   ├── 结构化展示各字段
│   ├── 支持编辑和保存
│   └── 添加历史版本
│
├── [高] 图片预览功能
│   ├── 点击放大
│   ├── 旋转/缩放
│   └── PDF 预览
│
└── [高] 状态反馈
    ├── Loading 动画
    ├── 进度条
    └── 错误提示

Week 2:
├── [中] 批量处理功能
│   ├── 批量 OCR
│   ├── 批量提取
│   └── 进度显示
│
├── [中] 数据导出
│   ├── 导出 PDF 报告
│   ├── 导出 Excel
│   └── 导出 JSON
│
└── [中] AI 优化
    ├── 改进 OCR prompt
    ├── 添加更多提取字段
    └── 失败重试机制
```

### Phase 3: 多端支持 (预计 2-4 周)

**目标：开发移动端和同步服务**

```
Week 3-4: Android App
├── 项目搭建 (React Native / Flutter)
├── 拍照上传功能
├── 相册导入
├── 离线缓存
└── 与服务器同步

Week 5-6: macOS App
├── Tauri 桌面应用
├── 扫描仪集成
├── 本地存储优化
└── 菜单栏快捷操作

Week 7-8: 同步服务
├── 同步服务开发
├── 变更检测
├── 冲突合并
├── 离线队列
└── Docker 部署方案
```

### Phase 4: 商业化 (按需)

**目标：产品化，支持商业化运营**

```
├── 用户系统
│   ├── 注册/登录
│   ├── 多租户支持
│   └── 权限管理
│
├── 数据安全
│   ├── 端到端加密
│   ├── 数据审计
│   └── 合规认证
│
├── 云端部署
│   ├── 公有云部署
│   ├── 自动扩缩容
│   └── 监控告警
│
└── 商业模式
    ├── 免费版 (本地)
    ├── Pro 版 (云同步)
    └── 企业版 (私有部署)
```

## 更新日志

### v0.3.0 (2026-03-24)
- 提取结果展示优化
  - 独立模态框展示 AI 提取结果
  - 卡片式布局展示诊断、主诉、治疗、药物、复诊建议等

- 就诊记录编辑功能
  - 支持编辑医院、科室、医生、主诉、诊断、治疗、摘要、备注
  - 添加 PUT API 更新就诊记录

### v0.2.0 (2026-03-23)
- 成员健康信息支持
  - 既往病史：慢性病、手术史、住院史、重大疾病史
  - 家族病史
  - 长期用药记录

  - 基础体征：身高、体重、BMI 计算
- 图片预览功能：点击放大、缩放

### v0.1.0 (2026-03-22)
- 初始版本发布
- CLI 工具完成
- Web UI 基础功能
- 支持 qwen3.5-plus OCR
- 支持 kimi-k2.5 提取
- 快速录入功能：上传图片自动识别预填表单
- 家庭成员排序：本人优先，其他按名字字母序
- 家庭成员默认选中：自动选中第一个成员并高亮
- 搜索筛选：支持按成员、医院、医生筛选就诊记录
